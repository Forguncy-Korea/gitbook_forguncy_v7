# 작업유형 - 삭제

리스트뷰 업데이트명령에서 작업 유형이 삭제인 경우 리스트뷰의 데이터를 삭제할 수 있지만 데이터베이스에 즉시 커밋되지는 않습니다.

## 삭제&#x20;

작업 유형이 삭제인 경우 다음 설정이 필요합니다.

* 대상 리스트뷰: 리스트 작업 명령을 실행할 때 업데이트되는 페이지에서 리스트뷰 선택해야 합니다.
* 레코드 업데이트: 삭제할 레코드를 설정합니다. 리스트뷰에서 선택한 행, 특정 조건을 충족하는 리스트뷰의 행 또는 리스트뷰의 모든 행을 선택할 수 있습니다. 이 설정은 고급 설정-작업별 레코드를 참조하는 데 사용되는 데이터 테이블 작업 명령의 업데이트 레코드와 동일합니다.

예를 들어 주문 목록 페이지에서 삭제 버튼을 클릭하여 리스트뷰의 데이터를 삭제하고 리스트 작업 명령을 사용할 수 있습니다.\
업데이트 형식을 \[삭제]로 선택하고 데이터를 업데이트할 테이블을 선택한 후 다음 그림과 같이 레코드를 기본 리스트뷰에서 선택한 행으로 업데이트합니다.



업을 \[삭제]로 선택하고 데이터를 업데이트할 테이블을 선택한 후 다음 그림과 같이 레코드를 기본 \[테이블]으로 선택한 행으로 업데이트합니다.

![](<../../../.gitbook/assets/image (296).png>)

실행 후 두 번째 데이터 행을 선택하고 삭제를 클릭하면 페이지의 두 번째 데이터 행이 삭제되었지만 데이터 테이블의 데이터가 삭제되지 않았음을 알 수 있습니다.
