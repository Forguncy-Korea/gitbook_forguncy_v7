# 작업유형 - 추가

리스트뷰 작업 명령에서 작업 유형이 \[추가]인 경우 리스트에 데이터를 추가할 수 있지만 데이터베이스에 즉시 커밋되지는 않습니다.

![](<../../../.gitbook/assets/image (376).png>)

## 추가&#x20;

작업 유형이 추가인 경우 다음을 설정해야 합니다.

* 대상 리스트뷰: 리스트뷰 작업 명령을 실행할 때 리스트뷰에 데이터를 추가하려면 페이지에서 리스트뷰를 선택해야 합니다.
* 열 및 값: 리스트뷰에 데이터를 추가할 열과 데이터를 설정합니다.
  * 열: 수식을 사용하여 리스트 템플릿 행의 셀을 참조하거나 열 이름을 입력할 수 있습니다.
  * 데이터: 추가된 열에 대한 데이터를 설정합니다. 데이터는 고정 값, 수식 또는 키워드일 수 있습니다.

예를 들어 주문 목록 페이지에서 추가 버튼을 클릭하여 리스트뷰에 데이터를 추가하면 리스트뷰 업데이트 명령 만들기  명령을 사용할 수 있습니다.<br>

작업 유형을 \[추가]로 선택하고 데이터를 추가할 리스트뷰를 선택한 후 리스트뷰 열 및 데이터 영역 아래의 \[새 열]을 클릭하고 다음 그림과 같이 데이터를 추가할 리스트뷰의 열과 해당 데이터를 선택합니다.

![](<../../../.gitbook/assets/image (363).png>)

실행 후 주문 목록 페이지에서 추가를 클릭하면 주문 A15가 리스트뷰에 추가되고 해당 열의 데이터가 이전 단계에서 설정한 고정 값 또는 수식의 값임을 확인할 수 있습니다. 그러나 데이터 테이블에 새로 추가된 데이터가 없습니다.
