# 작업 유형 - 리스트뷰 반영

리스트뷰 작업 명령을 사용하는 경우 추가, 업데이트, 삭제, 테이블에 데이터 추가, 업데이트 또는 삭제로 작업 유형을 설정할 수 있지만 데이터가 데이터베이스에 즉시 커밋되지는 않습니다.

작업 유형을 리스트뷰반영으로 설정하면 데이터 편집, 데이터 추가 및 데이터 삭제를 포함하여 테이블의 데이터 변경 내용을 데이터베이스에 커밋할 수 있습니다.

## 리스트뷰 반영&#x20;

양식의 데이터 추가, 편집 및 삭제와 같은 모든 변경 내용을 데이터베이스에 커밋할 수 있는 작업 유형을 \[리스트뷰 반영]으로 설정합니다.

예를 들어 주문 목록 페이지에서 추가 버튼을 클릭하여 테이블에 데이터를 추가할 수 있지만 이 시점에서 추가된 데이터는 작업 유형-추가를 참조하여 데이터베이스에 즉시 제출할 수 없습니다.

테이블에 데이터 변경 내용을 데이터베이스에 제출하려면 다음 그림과 같이 리스트뷰 반영 버튼을 추가하고 명령을 리스트뷰반영으로 설정하고 작업 유형을 테이블 제출로 설정하고 다음 그림과 같이 데이터를 제출할 테이블을 선택해야 합니다.

![](<../../../.gitbook/assets/image (350).png>)

실행 후 추가버튼을 클릭하면 테이블에 데이터가 추가되고 리스트 뷰반영 버튼을 클릭하여 새로 추가된 데이터를 데이터베이스에 커밋합니다.
