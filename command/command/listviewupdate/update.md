# 작업 유형 - 편집

리스트 작업 명령에서 작업 유형이 업데이트인 경우 테이블의 데이터를 업데이트할 수 있지만 데이터테이블 즉시 커밋되지는 않습니다

## 편집&#x20;

작업 유형이 업데이트인 경우 다음 설정이 필요합니다.

* 대상 리스트뷰: 리스트뷰 업데이트 명령만들기 실행할 때 편집되는 페이지에서 리스트뷰를 선택해야 합니다.
* 열 및 값: 데이터를 업데이트 리스트뷰의 열과 데이터를 설정합니다.
  * 열: 수식을 사용하여 테이블 템플릿 행의 셀을 참조하거나 열 이름을 입력할 수 있습니다.
  * 값: 업데이트된 열에 대한 데이터를 설정합니다. 데이터는 고정 값, 수식 또는 키워드일 수 있습니다.
* 업데이트할 행: 업데이트할 행을 설정합니다. 리스트뷰에서 선택한 행, 행이 리스트뷰의 조건을 충족함, 리스트뷰의 모든 행 및 리스트의 현재 행을 선택할 수 있습니다. 여기서 리스트뷰에서 선택한 행, 특정 조건을 충족하는 리스트뷰의 행, 리스트뷰에 모든 행에 대한 설정은 데이터 테이블 작업 명령의 업데이트 레코드와 동일합니다. 리스트뷰의 현재 행은 테이블에서 현재 선택한 셀이 있는 행입니다.

예를 들어 주문 목록 페이지에서 편집 버튼 클릭하여 리스트뷰의 데이터를 업데이트하고 리스트뷰업데이트 명을 사용할 수 있습니다.<br>

작업 유형을 \[편집]으로 선택하고 데이터를 업데이트할 리스트뷰를 선택한 후 리스트뷰 열 및 데이터 영역 아래의 \[새 열]을 클릭하고 리스트뷰에서 데이터를 업데이트할 열과 해당 데이터를 선택하고 다음 그림과 같이 기본 리스트뷰에서 선택한 행으로 값을 업데이트합니다.

![](<../../../.gitbook/assets/image (317).png>)

실행 후 두 번째 데이터 행을 선택하고 업데이트를 클릭하면 두 번째 데이터 행의 주문 날짜가 오늘 날짜로 업데이트되고 구매자가  업데이트되었음을 알 수 있습니다. 그러나 데이터 테이블의 데이터는 업데이트되지 않습니다.
