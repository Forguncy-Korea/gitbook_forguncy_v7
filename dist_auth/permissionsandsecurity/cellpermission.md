# 셀 권한

각 역할의 사용자가 액세스할 수 있는 요소를 정확하게 제어하기 위해 셀 유형이 설정된 셀에 대한 사용 권한을 설정할 수 있도록 지원합니다.

셀 유형이 설정된 셀을 선택하고 셀 설정에서 셀 권한을 클릭하면 셀 유형의 보기, 사용 권한 설정 대화상자가 나타납니다.

셀에 대한 사용하기 권한 및 보이기 권한을 설정할 수 있습니다.

* 사용하 권한: 사용자가 이 셀을 사용할 수 있는지 여부입니다. 셀 유형이 버튼인 셀과 같이 사용자가 버튼을 클릭하고 적절한 명령을 실행할 수 있는지 여부, 셀 유형이 텍스트 상자인 셀로, 사용자가 텍스트 상자에 문자를 입력할 수 있는지 여부입니다.
* 보이 권한: 사용자가 셀을 볼 수 있는지 여부입니다.



셀 유형의 보기, 사용 권한 설정 대화상자에서 사용을 선택한 다음 권한 부여 역할을 선택합니다.

셀 권한 설정을 지원하는 셀 유형에는 텍스트 상자, 여러 줄 텍스트 상자, 체크박스, 체크박스 그룹, 라디오 버튼, 콤보 상자, 숫자, 날짜, 시간, 그림 업로드, 첨부 파일, 사용자 선택 상자가 포함됩니다.



예를 들어 주문 목록 페이지에서 버튼 추가에 대한 셀 권한을 설정하고 사용하기 권한에서 "활성화 설정"을 선택한 다음 권한 부여 역할을 관리자로 선택합니다.&#x20;

<br>

![](<../../.gitbook/assets/image (280).png>)

보이기 권한에서 활성화 사용을 선택하고 권한 부여 역할을 로그인 사용자로 선택합니다.

![](<../../.gitbook/assets/image (1047).png>)

셀 권한을 설정한 후 리본 메뉴 모음에서 \[배포->서버배포]를 선택하고 배포 설정 대화 상자에서 다음 \[로컬의 페이지 접근 권한과 UI 사용 권한 설정을 서버에 덮어쓰기]를 선택합니다.

![](<../../.gitbook/assets/image (1063).png>)

게시 후, 관리자 역할인 유재석으로 로그인을 하면  버튼을 사용할 수 있습니다.

![](<../../.gitbook/assets/image (214).png>)

관리자역할이 아닌 조세호로 로그인하면 버튼은 볼 수 있지만 사용할 수 없다.

![](<../../.gitbook/assets/image (195).png>)

관리 콘솔로 이동합니다. 앱 관리를 선택하고 모든 앱 목록에서 앱 이름을 클릭하여 앱 설정 인터페이스로 이동합니다.

셀 권한을 클릭하면 셀 사용 권한이 설정된 모든 셀과 셀 권한 설정을 직접 변경할 수 있는 소유 페이지를 볼 수 있습니다.

예를 들어 버튼에 대한 사용하기 권한을 로그인한 사용자가 액세스할 수 있도록 변경하고 설정 저장을 클릭합니다.

![](<../../.gitbook/assets/image (497).png>)

앱 이름을 클릭한 후 앱으로 이동하여 조세호로 로그인하면 조세호가 추가 버튼을 사용할 수 있습니다.![](https://help.grapecity.com.cn/download/thumbnails/72357001/image2019-9-23_11-42-2.png?version=1\&modificationDate=1648092600000\&api=v2)

![](<../../.gitbook/assets/image (236).png>)
